Add `read_existing` option to `ZoneFileProvider` so it populates from existing zone files on disk when used as a target. Default `false` preserves current behavior; opting in makes re-runs idempotent (no SOA serial bump when nothing changed). Both `_load_zone_file` and `zone_exists` honor the flag, so apex NS records are diffed truthfully against on-disk state — meaning the `RootNsChange` guard rearms (intentionally) when sources disagree with the existing delegation.
