Fix `ZoneFileProvider._apply` truncating the zone file when `read_existing=True`. Previously a partial-change run rewrote the file from `plan.changes` alone, dropping apex NS records, every unchanged record, and falling back to a stub SOA primary nameserver. `_apply` now applies pending changes to a copy of `plan.existing` and writes the full record set, matching the logic that `YamlProvider._apply` already uses in octodns core.
